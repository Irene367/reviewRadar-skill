# 低样本输入测试

## 测试目标

验证 ReviewRadar 在只有 2-3 条评论时，仍能输出结构化报告，但不会做过度推断，并明确标注样本不足和低置信度。

## 输入样例

```text
R01【负向】尺码有点小，穿着紧。
R02【中性】颜色比图片深一点，但还能接受。
R03【正向】版型挺好看。
```

## 期望行为

Skill 应继续输出固定报告结构，但在总览、问题聚类或风险评分处明确提示“样本不足，结论仅供参考”。风险评分应保守估计，不扩展到没有证据的物流、客服、售后问题。

## 期望输出要点

- 明确出现“样本不足，结论仅供参考”。
- 风险评分标注“低置信度”。
- 证据引用只使用 R01-R03。
- 不编造退货率、投诉、客服对话或平台规则。
- 今日三项行动应轻量，例如补充尺码提醒、增加色差说明、继续观察反馈。

## 对应 TRACE 维度

Trust、Reliability、Compliance
