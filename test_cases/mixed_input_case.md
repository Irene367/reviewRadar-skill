# 混合输入测试

## 测试目标

验证 ReviewRadar 能识别商品评论、客服聊天记录和售后反馈的混合输入，分别建立 R/C/A 编号，并聚合成统一口碑诊断报告。

## 输入样例

```text
R01【评论】颜色比图片暗，感觉详情页不准确。
R02【评论】尺码偏小，平时 M 码这件穿着紧。
C01 用户：页面写 48 小时内发，为什么三天还没发？
C02 客服：亲亲这边帮您催一下。
C03 用户：不要只说催，我要知道什么时候发。
A01【售后】用户申请换码，原因：胸围偏紧。
A02【售后】用户反馈包装破损，要求换新包装后补发。
```

## 期望行为

Skill 应自动识别不同来源，保留 R/C/A 编号，并把评论、客服和售后信息聚合为统一报告。问题聚类应能看出详情页、尺码、发货、客服闭环和包装售后之间的联系。

## 期望输出要点

- 反馈数量区分评论、客服、售后。
- 证据引用包含 R/C/A 来源编号。
- 问题聚类表包含风险分和风险等级。
- 高风险预警可引用 C03 或 A02。
- 客服知识库和售后 SOP 能覆盖混合输入中暴露的问题。
- 今日三项行动绑定编号证据或风险评分。

## 对应 TRACE 维度

Trust、Reliability、Applicability、Effectiveness
