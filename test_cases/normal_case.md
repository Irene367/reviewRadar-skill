# 正常评论输入测试

## 测试目标

验证 ReviewRadar 在 15-25 条常规商品评论输入下，能稳定输出完整口碑诊断报告，并包含证据编号、风险评分、置信度、客服知识库更新包、详情页改写、售后 SOP 和今日三项优先行动。

## 输入样例

```text
R01【正向】版型很好看，上身显瘦。
R02【负向】平时 M 码，这件 M 码太紧。
R03【负向】发货慢，说 48 小时内发，第四天才动。
R04【负向】颜色比图片暗很多。
R05【中性】价格还可以，但面料没有想象中厚。
R06【负向】客服回复慢，只说帮我催。
R07【负向】包装袋破了。
R08【正向】长度适合小个子。
R09【负向】详情页没写清楚尺码，买错很麻烦。
R10【负向】线头比较多。
R11【中性】可以通勤穿，但容易皱。
R12【负向】物流揽收后两天没动。
R13【负向】客服态度可以，但问题没解决。
R14【正向】洗了一次没有掉色。
R15【强负向】发货慢、颜色不对、客服也没给方案，不会再买。
```

## 期望行为

Skill 应输出完整结构化报告，不停留在评论摘要。报告应能识别尺码、发货、色差、客服、包装、质量等问题，并将问题转为可执行动作。

## 期望输出要点

- 包含 1-9 节完整报告结构。
- 问题聚类表包含“风险分”“置信度”和“风险等级”。
- 证据引用保留 `Rxx` 编号。
- 至少一个高风险问题进入 P0。
- 客服知识库更新包包含触发关键词和人工介入条件。
- 详情页优化包含可直接复制的改写示例。
- 今日三项优先行动均为店主本人可独立执行。

## 对应 TRACE 维度

Trust、Reliability、Applicability、Effectiveness
