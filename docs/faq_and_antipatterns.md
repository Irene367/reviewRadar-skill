# FAQ 与反模式说明

## FAQ

### 1. 我只有几条评论，也能用吗？

可以。少于 5 条评论时会输出低置信度简版报告，只适合初步判断，不建议据此做重大经营决策。

### 2. 我需要上传 Excel 吗？

不需要。第一次使用可以直接粘贴 5-10 条评论。Excel / CSV 只是进阶输入方式。

### 3. 评论里有手机号、地址、订单号怎么办？

建议先脱敏。如果用户没有脱敏，Skill 应自动忽略或用 `[已脱敏]` 替代，不在输出中复述敏感信息。

### 4. 为什么风险分不是精确计算？

风险分是基于评论证据的辅助估算，不代表真实退货率、投诉率或转化率。它用于帮助一人店主判断优先级。

### 5. 我只想生成差评回复，需要完整报告吗？

不需要。用户可以触发单任务模式，只输出差评回复相关内容。

### 6. 我能用它分析淘宝、抖店、小红书、Shopify 或美团评论吗？

可以，只要数据是用户主动提供、商家后台导出或手动粘贴的文本。本 Skill 不自动抓取平台数据，也不绕过平台权限。

### 7. 它能帮我删除差评、刷好评或控评吗？

不能。Skill 会拒绝刷评、控评、删评、诱导好评等违规请求，并提供合规的回复、售后和体验优化方案。

### 8. 如果输入内容很乱怎么办？

Skill 会先做输入质量检查，尽量识别评论、客服、售后等来源；如果无法判断，会标注低置信度并建议补充数据。

### 9. 如果我的商品不是女装，还能用吗？

可以。Skill 会先识别品类，并根据品类调整问题聚类维度。如果品类无法判断，会使用通用维度：质量、描述一致性、履约、包装、客服、售后、价格预期。

### 10. 第一次使用应该怎么开始？

建议先使用 3 分钟上手 Demo，粘贴 5 条评论，触发“简版模式”，先看 Top 3 问题和今日三项行动。

## 不建议这样使用

| 反模式 | 为什么不建议 | 推荐做法 |
|---|---|---|
| 一上来上传大量混乱文本 | 会增加识别难度，输出可能不稳定 | 先用 5-10 条评论跑简版 |
| 把不同商品的评论混在一起 | 问题归因会变差 | 按商品或品类分批分析 |
| 只看风险分，不看证据 | 风险分是估算，不是精确经营指标 | 同时查看 R/C/A 证据编号 |
| 要求生成刷评、删评、控评方案 | 违反平台规则和诚信经营原则 | 生成合规回复与售后改进方案 |
| 用 2 条评论做重大经营决策 | 样本不足，结论不稳 | 补充更多评论和客服记录 |
| 把隐私信息原样上传 | 存在隐私风险 | 先脱敏或让 Skill 自动忽略 |
