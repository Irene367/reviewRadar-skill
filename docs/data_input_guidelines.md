# ReviewRadar 数据输入规范

ReviewRadar 只分析用户主动提供的数据。当前版本不自动连接平台 API，不抓取网页，不绕过登录验证，不采集非授权数据。

## 支持的数据提供方式

### 粘贴评论文本

可以直接粘贴商品评论、追加评论、差评、中评或好评。例如：

```text
评论 1：尺码偏小，平时 M 码这次有点紧。
评论 2：客服态度挺好，但发货等太久。
```

### 上传或粘贴 CSV / Excel 导出的文本

可以从商家后台主动导出评论或售后表格，再复制其中的文本内容。建议保留字段名，例如：

```text
评价时间,商品,评分,评论内容,售后状态
2026-05-30,通勤连衣裙,2,颜色比图片暗，尺码偏小,申请换货
```

如果表格中包含姓名、手机号、地址、订单号等隐私字段，请先删除对应列。

### 粘贴客服聊天记录

可以粘贴用户和客服的对话。建议保留说话人，但不需要保留用户账号、电话、地址或订单号。例如：

```text
用户：页面写 48 小时发货，为什么三天还没发？
客服：抱歉，这边帮您催一下仓库。
```

### 混合输入

可以把评论、差评、售后反馈和客服记录放在一起。ReviewRadar 会先按来源和问题类型粗分，再生成统一的口碑诊断报告。

## 脱敏建议

输入前建议删除或替换以下信息：

- 姓名
- 手机号
- 收货地址
- 订单号
- 账号 ID
- 物流单号
- 身份证、支付账号等高敏感信息

如果用户已经粘贴了敏感信息，ReviewRadar 应在分析中忽略，不应复述或用于判断。

## 样本量说明

样本越多，聚类和优先级判断越稳定。若只有少量评论或一段聊天记录，ReviewRadar 仍可输出报告，但必须提示“样本不足，结论仅供参考”。商家应结合真实订单量、退换货率、客服工单和实际库存履约情况人工确认。

## 当前版本边界

- 不自动连接淘宝、京东、拼多多、抖店、小红书店铺、Shopify、美团/大众点评等平台 API。
- 不做网页爬虫。
- 不绕过平台登录或权限验证。
- 不自动下载、采集或保存用户隐私数据。
- 未来可通过商家授权 API 接入多平台，但 MVP 阶段不实现该能力。
