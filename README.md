# ReviewRadar 店铺口碑雷达

第一次使用、查看 FAQ、完整案例和测试用例，请先阅读 [START_HERE.md](START_HERE.md)。

PriceRadar 解决一人电商公司的外部竞争问题：竞品怎么变，我该怎么定价。  
ReviewRadar 解决一人电商公司的内部口碑问题：用户为什么不满意，我该怎么优化。  

两个 Skill 共同组成「一人电商公司 AI 经营双雷达」：

- 外看竞品：PriceRadar
- 内看口碑：ReviewRadar

ReviewRadar 不是普通评论总结工具，而是将评论、差评和客服反馈转化为“风险判断 + 证据追溯 + 运营交付物 + 今日行动”的一人店铺口碑运营 Skill。


## 项目定位

ReviewRadar 是一个平台无关的评论、差评、客服反馈分析 Skill，面向一人电商公司、独立卖家和小商家，把分散的商品评论、差评、客服聊天记录、售后反馈，转化为结构化的口碑诊断报告和可执行运营动作。

它把用户反馈转换成差评回复、客服知识库更新包、商品详情页改写示例、售后 SOP 和今日三项优先行动，帮助一人公司 OPC 用一个人完成客服主管、运营、品控和文案团队的协作流程。

## 目标用户

- 一人电商公司和独立卖家
- 小型淘宝、京东、拼多多、抖店、小红书店铺、Shopify 店主
- 本地生活商家、团购商家和内容电商团队
- 没有专职客服主管、运营分析师、品控负责人和文案团队的小商家

## 解决的问题

- 评论和客服记录分散，卖家难以每天复盘
- 差评背后的真实原因不清楚，只能被动补救
- 高频问题没有沉淀进 FAQ 和客服知识库
- 商品详情页承诺与用户实际感知存在偏差
- 售后处理依赖临场发挥，缺少稳定 SOP
- 一人公司没有足够人手做口碑、运营、品控和文案协同

## 核心能力

- R/C/A 证据编号和证据追溯
- 商品评论和差评归因分析
- 五维风险评分：出现频次、情绪强度、转化影响、售后成本、扩散风险
- 客服聊天记录复盘
- 用户痛点聚类与风险等级判断
- 高风险差评预警
- 合规、克制、可执行的差评回复话术
- 客服知识库更新包
- 商品详情页优化建议和可复制改写示例
- 售后 SOP 生成
- 每日 / 每周口碑运营日报
- 今日三项优先行动

### 6. 低样本快速判断
我只有以下几条评论，请帮我做初步口碑判断，并标注低置信度和不确定结论。

## 最小完整案例

### 输入示例

```text
R01【负向】等了五天才发货，客服只说帮忙催一下。
R02【负向】颜色比图片深很多，有点失望。
R03【中性】包装有点破，但商品本身还可以。
R04【负向】客服态度不错，但没有解决问题。
R05【正向】版型好看，价格也能接受。
```

### 输出摘要

- Top 1：发货慢 / 客服无闭环，风险等级高，置信度中。
- Top 2：颜色预期不一致，风险等级中，置信度中。
- Top 3：包装破损，风险等级中，置信度低。
- 今日行动：人工跟进发货异常、补充颜色说明、新增包装破损 FAQ。
更多常见问题和不建议用法见 [FAQ 与反模式说明](docs/faq_and_antipatterns.md)。

## 输出模式

ReviewRadar 支持两种输出模式：

### 简版模式

适合用户每天快速查看。只输出：

1. 本次口碑总览
2. Top 3 用户问题聚类
3. 高风险差评预警
4. 今日三项优先行动

触发方式：

- 用户说“简版”
- 用户说“快速看一下”
- 用户说“只要重点”
- 用户说“今天先做什么”

### 完整版模式

适合深度诊断、周报复盘、比赛 Demo 或店铺运营优化。输出完整 9 个模块：

1. 本次口碑总览
2. 用户问题聚类
3. 高风险差评预警
4. 差评回复话术
5. 客服知识库更新包
6. 商品详情页优化建议
7. 售后 SOP 建议
8. 今日口碑运营日报
9. 今日三项优先行动

触发方式：

- 用户说“完整报告”
- 用户说“深度诊断”
- 用户说“输出完整 9 个模块”
- 用户没有指定模式时，默认使用完整版模式

## 使用方式

1. 从商家后台主动导出评论、售后或客服记录，或手动复制相关文本。
2. 在输入前尽量删除姓名、手机号、地址、订单号等敏感信息。
3. 把评论列表、差评列表、客服聊天记录、CSV/Excel 导出文本或混合文本粘贴给 ReviewRadar。
4. ReviewRadar 自动建立证据编号，输出固定结构的《店铺口碑诊断报告》。
5. 商家人工确认后，将建议应用到客服回复、FAQ、详情页和售后 SOP 中。

## 输出示例摘要

ReviewRadar 会输出：

- 本次口碑总览：反馈数量、情绪概览、口碑风险等级、最突出问题
- 用户问题聚类：问题类型、频次、风险分、置信度、风险等级、原文证据、优先级
- 高风险差评预警：原始反馈、风险原因、影响判断、建议动作

- 详情页优化建议：新增信息、修改描述、降低误解、展示方式、可复制改写示例
- 售后 SOP 和今日口碑运营日报

## 品类适配能力

ReviewRadar 不只适用于女装，也可用于美妆、食品、家居、数码配件、宠物用品、母婴用品、文创用品等一人电商场景。

不同品类的分析重点会自动调整：

| 品类 | 重点关注问题 |
|---|---|
| 女装 / 服饰 | 尺码、版型、色差、面料、发货、包装 |
| 美妆个护 | 肤质适配、过敏反馈、色号差异、气味、包装破损、保质期 |
| 食品饮料 | 口味、保质期、破损漏液、发货时效、包装完整性 |
| 家居用品 | 尺寸、材质、安装难度、耐用性、物流破损 |
| 数码配件 | 兼容性、做工、发热、使用稳定性、售后响应 |
| 宠物用品 | 适口性、安全性、尺寸适配、材质、异味 |
| 母婴用品 | 安全性、材质、适龄范围、使用便利性、售后保障 |
| 文创用品 | 印刷质量、颜色还原、包装、发货、做工细节 |

## 小众品类兜底

如果商品品类不在预设列表中，ReviewRadar 会使用通用品类诊断维度：

1. 描述一致性：用户收到的商品是否符合页面描述；
2. 质量与做工：是否存在破损、瑕疵、耐用性问题；
3. 履约体验：发货、物流、包装是否符合预期；
4. 使用门槛：是否存在安装、学习、适配、操作困难；
5. 客服与售后：是否及时响应并形成闭环；
6. 价格预期：用户是否认为价格与体验匹配；
7. 风险信号：是否出现投诉、退款、公开差评或强烈情绪。

## 平台无关设计说明

ReviewRadar 不绑定淘宝、京东、拼多多、抖店、小红书店铺、Shopify、美团/大众点评等任何具体平台。MVP 阶段不接平台 API，不做网页爬虫，不绕过平台权限。数据来源只支持用户手动粘贴、用户主动导出的 CSV/Excel/文本内容，以及用户粘贴的客服聊天记录。

未来可以通过商家授权 API 接入多平台，但当前版本不实现任何外部 API、爬虫或自动登录逻辑。

## 合规说明

ReviewRadar 不采集用户隐私，不诱导刷评，不建议删除差评，不提供虚假宣传话术，不输出操纵平台评价的建议。若输入中出现姓名、手机号、地址、订单号等敏感信息，应先脱敏；分析时也应自动忽略。所有运营建议、售后承诺、赔付动作和公开回复都需要商家人工确认。

## TRACE 自评摘要

- Trust：结论绑定带编号的原始评论证据，避免无依据判断。
- Reliability：固定报告结构、风险评分规则和异常兜底机制，异常输入也能稳定输出并标注置信度。
- Applicability：面向一人电商公司每天都会遇到的评论、客服和售后场景。
- Compliance：不接非授权 API，不爬虫，不诱导刷评，不建议删差评。
- Effectiveness：把客服、运营、品控、文案协作压缩成一个人可执行的口碑运营工作流。
